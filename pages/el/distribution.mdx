# Πώς Λειτουργούν τα JetPack Tokens

Το σύστημα token του JetPack περιλαμβάνει **έκδοση και διανομή token με βάση τη δραστηριότητα**, ένα **σύστημα επιπέδου υπηρεσιών**, **ανταμοιβές χρηστών και μεθόδους αξιοποίησης token**, και έναν **μηχανισμό αγοράς και αναδιανομής token**. Αυτή η δομή παρέχει ένα **δίκαιο και αποδοτικό σύστημα ανταμοιβών** τόσο για τους χρήστες όσο και για τους παρόχους υπηρεσιών.

## Μηχανισμός Διανομής Token (Drop)

### Συνολική Προσφορά Token και Περίοδος Διανομής

- **Συνολική Προσφορά**: Το JetPack θα εκδώσει συνολικά **1 δισεκατομμύριο Jet** tokens.
- **Περίοδος Διανομής**: Τα tokens θα διανεμηθούν σε διάστημα **38 ετών**, με συνολικά **2,776 διανομές** που θα πραγματοποιηθούν στα ίδια διαστήματα με την **Εποχή Cardano** (1 εποχή = 5 ημέρες).

### Ποσότητα Διανομής Token ανά Εποχή

#### Υπολογισμός Ποσότητας Διανομής Βάσει Λογαριθμικής Συνάρτησης

Η ποσότητα των tokens που διανέμονται ανά εποχή έχει σχεδιαστεί να μειώνεται με την πάροδο του χρόνου. Μια **λογαριθμική συνάρτηση** χρησιμοποιείται για τον υπολογισμό της ποσότητας διανομής για κάθε εποχή. **1 Jet ισούται με 1,000,000 Pack**, οπότε οι ποσότητες διανομής υπολογίζονται μέχρι την **έκτη δεκαδική θέση** και πληρώνονται αναλόγως.

1. **Ορισμός της Λογαριθμικής Συνάρτησης $ f(t) $**:

   $$
   f(t) = \dfrac{1}{\ln(t + 1)}
   $$

   Όπου:

    - $ t $ είναι ο αριθμός της εποχής, $ t = 1, 2, 3, \dotsc, 2776 $.
    - $ \ln $ είναι η φυσική λογαριθμική συνάρτηση.

2. **Υπολογισμός του Αθροίσματος της Συνάρτησης Log $ f(t) $ σε Όλες τις Εποχές**:

   Το συνολικό άθροισμα των τιμών της συνάρτησης σε όλη την περίοδο διανομής $ S $ είναι:

   $$
   S = \sum_{t=1}^{2776} f(t) = \sum_{t=1}^{2776} \dfrac{1}{\ln(t + 1)}
   $$

3. **Υπολογισμός της Σταθεράς Κλιμάκωσης $ A $**:

   Η συνολική προσφορά token των **1 δισεκατομμυρίων Jet** διαιρείται με το άθροισμα $ S $ για να προσαρμοστεί η ποσότητα διανομής για κάθε εποχή:

   $$
   A = \dfrac{1,000,000,000}{S} = 2,412,005.7263929145
   $$

4. **Υπολογισμός της Ποσότητας Διανομής Token ανά Εποχή**:

   Η ποσότητα διανομής για κάθε εποχή είναι:

   $$
   \text{Ποσότητα Διανομής}_t = A \times f(t) = A \times \dfrac{1}{\ln(t + 1)}
   $$

5. **Παραδείγματα Ποσοτήτων Διανομής**:

   Υπολογισμός των ποσοτήτων διανομής για τις αρχικές εποχές:

    - **Εποχή 1**:

      $$
      \text{Ποσότητα Διανομής}_1 = 3,479,788.70 = A \times \dfrac{1}{\ln(1 + 1)} = A \times \dfrac{1}{\ln(2)}
      $$

    - **Εποχή 2**:

      $$
      \text{Ποσότητα Διανομής}_2 = 2,195,502.227 = A \times \dfrac{1}{\ln(2 + 1)} = A \times \dfrac{1}{\ln(3)}
      $$

    - **Εποχή 3**:

      $$
      \text{Ποσότητα Διανομής}_3 = 1,739,894.35 = A \times \dfrac{1}{\ln(3 + 1)} = A \times \dfrac{1}{\ln(4)}
      $$

   Η ποσότητα διανομής για κάθε εποχή καθορίζεται από την τιμή της λογαριθμικής συνάρτησης που αντιστοιχεί στον αριθμό της εποχής.

**Χαρακτηριστικά Διανομής**

- Δεδομένου ότι $ \ln(t + 1) $ αυξάνεται καθώς αυξάνεται το $ t $, ο παρονομαστής γίνεται μεγαλύτερος, προκαλώντας τη συνολική ποσότητα διανομής να μειώνεται σταδιακά.
- Η ποσότητα διανομής ανά εποχή μειώνεται με την πάροδο του χρόνου, πλησιάζοντας το μηδέν.
- Αυτός ο σχεδιασμός ενθαρρύνει τους πρώιμους συμμετέχοντες και αυξάνει τη σπανιότητα του token.

### Κριτήρια Διανομής Token ανά Υπηρεσία

#### Υπολογισμός Αναλογιών Μεταξύ Υπηρεσιών

Η αναλογία διανομής για κάθε υπηρεσία υπολογίζεται βάσει της βαθμολογίας απόδοσής της $ P_n $ σε σχέση με τη συνολική βαθμολογία απόδοσης $ P $:

$$
\text{Αναλογία για Υπηρεσία } n = \frac{P_n}{P}
$$

Όπου:

- $ P_n $: Βαθμολογία απόδοσης της υπηρεσίας $ n $.
- $ P $: Άθροισμα βαθμολογιών απόδοσης όλων των υπηρεσιών.

#### Υπολογισμός Ποσότητας Διανομής ανά Υπηρεσία

Η ποσότητα των tokens που λαμβάνει η υπηρεσία $ n $ σε κάθε εποχή $ S_n $ είναι:

$$
S_n = \min\left(f_{\text{total}}(t) \times \left( \frac{P_n}{P} \right), \text{Μέγιστη Διανομή Token για την Υπηρεσία } n\right)
$$

- $ f_{\text{total}}(t) $: Προσαρμοσμένη συνολική ποσότητα διανομής για την εποχή.
- $ \text{Μέγιστη Διανομή Token για την Υπηρεσία } n $: Μέγιστη ποσότητα token που διατίθεται βάσει του επιπέδου της υπηρεσίας (δείτε παρακάτω τη μέγιστη διανομή token βάσει επιπέδου).

#### Διαχείριση Μη Διανεμημένων Tokens

##### Μεταφορά Μη Διανεμημένων Tokens

- Τα μη διανεμημένα tokens από κάθε εποχή **προστίθενται** στη συνολική ποσότητα διανομής για την επόμενη εποχή.
- Αυτό εξασφαλίζει αποτελεσματική διανομή token ακόμα και όταν η αρχική συμμετοχή των υπηρεσιών είναι χαμηλή.

##### Υπολογισμός Προσαρμοσμένης Συνολικής Ποσότητας Διανομής ανά Εποχή

Η συνολική ποσότητα διανομής για την επόμενη εποχή είναι:

$$
f_{\text{total}}(t+1) = f(t+1) + \text{Μη Διανεμημένα Tokens από την Προηγούμενη Εποχή}
$$

- $ f_{\text{total}}(t+1) $: Προσαρμοσμένη συνολική ποσότητα διανομής για την εποχή $ t+1 $.
- $ f(t+1) $: Ποσότητα διανομής βάσει της βασικής συνάρτησης διανομής.
- **Μεταφερόμενα Μη Διανεμημένα Tokens**: Αριθμός tokens που δεν διανεμήθηκαν στην εποχή $ t $.

#### Υπολογισμός Βαθμολογίας Απόδοσης ($ P_n $) ανά Υπηρεσία

Η βαθμολογία απόδοσης κάθε υπηρεσίας υπολογίζεται βάσει των παρακάτω μετρικών. Το βάρος κάθε μετρικής μπορεί να προσαρμοστεί μέσω διακυβέρνησης.

1. **Κατανάλωση Token JetPack ($ M_{1n} $)**
    - **Βάρος**: $ w_1 = 50\% $
    - **Περιγραφή**: Συνολική ποσότητα tokens JetPack που καταναλώθηκαν από χρήστες εντός της υπηρεσίας.

2. **Αριθμός Ψήφων Degen Bunny ($ M_{2n} $)**
    - **Βάρος**: $ w_2 = 30\% $
    - **Περιγραφή**: Αριθμός ψήφων που έλαβαν από OG Degen Bunnies.

3. **Αριθμός Ενεργών Χρηστών ($ M_{3n} $)**
    - **Βάρος**: $ w_3 = 20\% $
    - **Περιγραφή**: Αριθμός μοναδικών χρηστών που είχαν πρόσβαση και ήταν ενεργοί στην υπηρεσία κατά τη διάρκεια της εποχής.

Η βαθμολογία απόδοσης κάθε υπηρεσίας $ P_n $ υπολογίζεται ως εξής:

$$
P_n = w_1 \times \left( \frac{M_{1n}}{M_{1\text{total}}} \right) + w_2 \times \left( \frac{M_{2n}}{M_{2\text{total}}} \right) + w_3 \times \left( \frac{M_{3n}}{M_{3\text{total}}} \right)
$$

Όπου:

- $ M_{1n} $: Κατανάλωση token JetPack της υπηρεσίας $ n $.
- $ M_{1\text{total}} $: Συνολική κατανάλωση token JetPack σε όλες τις υπηρεσίες.
- $ M_{2n} $: Αριθμός ψήφων Degen Bunny για την υπηρεσία $ n $.
- $ M_{2\text{total}} $: Συνολικός αριθμός ψήφων Degen Bunny σε όλες τις υπηρεσίες.
- $ M_{3n} $: Αριθμός ενεργών χρηστών της υπηρεσίας $ n $.
- $ M_{3\text{total}} $: Συνολικός αριθμός ενεργών χρηστών σε όλες τις υπηρεσίες.

**Παράδειγμα**:

- **Υπηρεσία A**:
    - $ M_{1A} = 500 $
    - $ M_{2A} = 150 $
    - $ M_{3A} = 200 $
- **Υπηρεσία B**:
    - $ M_{1B} = 300 $
    - $ M_{2B} = 200 $
    - $ M_{3B} = 100 $
- **Υπηρεσία C**:
    - $ M_{1C} = 200 $
    - $ M_{2C} = 150 $
    - $ M_{3C} = 700 $

**Σύνολα**:

- $ M_{1\text{total}} = 500 + 300 + 200 = 1000 $
- $ M_{2\text{total}} = 150 + 200 + 150 = 500 $
- $ M_{3\text{total}} = 200 + 100 + 700 = 1000 $

**Βάρη**:

- $ w_1 = 0.5 $
- $ w_2 = 0.3 $
- $ w_3 = 0.2 $

**Υπολογισμός Βαθμολογίας Απόδοσης για την Υπηρεσία A**:

$$
\begin{align*}
P_A &= 0.5 \times \left( \frac{500}{1000} \right) + 0.3 \times \left( \frac{150}{500} \right) + 0.2 \times \left( \frac{200}{1000} \right) \\
&= 0.5 \times 0.5 + 0.3 \times 0.3 + 0.2 \times 0.2 \\
&= 0.25 + 0.09 + 0.04 \\
&= 0.38
\end{align*}
$$

**Υπολογισμός Βαθμολογίας Απόδοσης για την Υπηρεσία B**:

$$
\begin{align*}
P_B &= 0.5 \times \left( \frac{300}{1000} \right) + 0.3 \times \left( \frac{200}{500} \right) + 0.2 \times \left( \frac{100}{1000} \right) \\
&= 0.5 \times 0.3 + 0.3 \times 0.4 + 0.2 \times 0.1 \\
&= 0.15 + 0.12 + 0.02 \\
&= 0.29
\end{align*}
$$

**Υπολογισμός Βαθμολογίας Απόδοσης για την Υπηρεσία C**:

$$
\begin{align*}
P_C &= 0.5 \times \left( \frac{200}{1000} \right) + 0.3 \times \left( \frac{150}{500} \right) + 0.2 \times \left( \frac{700}{1000} \right) \\
&= 0.5 \times 0.2 + 0.3 \times 0.3 + 0.2 \times 0.7 \\
&= 0.10 + 0.09 + 0.14 \\
&= 0.33
\end{align*}
$$

Έτσι, οι βαθμολογίες απόδοσης είναι:

- **Υπηρεσία A**: $ P_A = 0.38 $
- **Υπηρεσία B**: $ P_B = 0.29 $
- **Υπηρεσία C**: $ P_C = 0.33 $

1. **Υπολογισμός Βαθμολογιών Απόδοσης**

   Χρησιμοποιήστε την προηγουμένως καθορισμένη μέθοδο για να υπολογίσετε τη βαθμολογία απόδοσης κάθε υπηρεσίας $ P_n $.

2. **Καθορισμός Κατατάξεων Υπηρεσιών**

   Κατατάξτε όλες τις υπηρεσίες σε **φθίνουσα σειρά** βάσει των βαθμολογιών απόδοσής τους.

3. **Υπολογισμός Σωρευτικών Κορυφαίων Ποσοστών**

   Για κάθε υπηρεσία, υπολογίστε το σωρευτικό κορυφαίο ποσοστό:

   $$
   \text{Σωρευτικό Κορυφαίο Ποσοστό (\%)} = \left(1 - \frac{\text{Κατάταξη} - 1}{N} \right) \times 100\%
   $$

   Όπου:

    - $ \text{Κατάταξη} $: Η κατάταξη της υπηρεσίας (ξεκινώντας από το 1).
    - $ N $: Συνολικός αριθμός υπηρεσιών.

4. **Ανάθεση Επιπέδων**

   Αναθέστε επίπεδα βάσει των σωρευτικών κορυφαίων ποσοστών:

    - **Επίπεδο 1**: Σωρευτικό κορυφαίο ποσοστό > 96%
    - **Επίπεδο 2**: 89% < Σωρευτικό κορυφαίο ποσοστό ≤ 96%

    - **Επίπεδο 4**: 60% < Σωρευτικό κορυφαίο ποσοστό ≤ 77%
    - **Επίπεδο 5**: 40% < Σωρευτικό κορυφαίο ποσοστό ≤ 60%
    - **Επίπεδο 6**: 23% < Σωρευτικό κορυφαίο ποσοστό ≤ 40%
    - **Επίπεδο 7**: 11% < Σωρευτικό κορυφαίο ποσοστό ≤ 23%
    - **Επίπεδο 8**: 4% < Σωρευτικό κορυφαίο ποσοστό ≤ 11%
    - **Επίπεδο 9**: Σωρευτικό κορυφαίο ποσοστό ≤ 4%

### Παράδειγμα Σεναρίου

#### Εποχή 1 ($ t = 1 $)

- **Βασική Ποσότητα Διανομής**: $ f(1) = 3,479,788.70 \text{ Jet} $
- **Προσαρμοσμένη Συνολική Ποσότητα Διανομής**: $ f_{\text{total}}(1) = 3,479,788.70 \text{ Jet} $

**Υπολογισμός Ποσότητας Διανομής ανά Υπηρεσία**

Βάσει των βαθμολογιών απόδοσης και των επιπέδων:

- **Συνολική Βαθμολογία Απόδοσης $ P $**: $ P = P_A + P_B + P_C = 0.38 + 0.29 + 0.33 = 1.00 $

    - Αναλογία: $ \frac{P_A}{P} = \frac{0.38}{1.00} = 0.38 $
    - Ποσότητα Διανομής:

      $$
      S_A = \min\left( 3,479,788.70 \times 0.38, 30,000 \right) = \min(1,322,319.71, 30,000) = 30,000 \text{ Jet}
      $$

    - Αναλογία: $ \frac{P_C}{P} = \frac{0.33}{1.00} = 0.33 $

      $$
      S_C = \min\left( 3,479,788.70 \times 0.33, 15,000 \right) = \min(1,148,330.27, 15,000) = 15,000 \text{ Jet}
      $$

3. **Υπηρεσία B**:

    - Αναλογία: $ \frac{P_B}{P} = \frac{0.29}{1.00} = 0.29 $
    - Ποσότητα Διανομής:

      $$
      S_B = \min\left( 3,479,788.70 \times 0.29, 7,000 \right) = \min(1,009,138.72, 7,000) = 7,000 \text{ Jet}

## Ανταμοιβές Χρηστών και Χρήση Token

### **Μηχανισμός Ανταμοιβής Χρηστών**

- **Ανταμοιβές Βάσει Δραστηριότητας**: Οι χρήστες λαμβάνουν tokens JetPack όταν εκτελούν **προκαθορισμένες δραστηριότητες** (π.χ., δημιουργία περιεχομένου, παιχνίδι) εντός μιας υπηρεσίας.
- **Κριτήρια Ανταμοιβής Token**: Η ποσότητα των tokens που ανταμείβονται για κάθε δραστηριότητα **καθορίζεται μέσω διαβούλευσης** μεταξύ της υπηρεσίας και του διαχειριστή του οικοσυστήματος JetPack.

### **Μέθοδοι Χρήσης Token**

1. **Εντός της Υπηρεσίας**:
    - **Πρόσβαση σε Premium Χαρακτηριστικά**: Χρησιμοποιήστε tokens για πρόσβαση σε πληρωμένα χαρακτηριστικά ή περιεχόμενο εντός της υπηρεσίας.
    - **Αγορά Αγαθών**: Αγοράστε ψηφιακά αγαθά ή αντικείμενα εντός της υπηρεσίας χρησιμοποιώντας tokens.

2. **Χρησιμοποιώντας την Υπηρεσία Ανταλλαγής Πορτοφολιού JetPack**:
    - **Πωλήσεις Token**: Οι χρήστες μπορούν να ανταλλάξουν τα tokens τους για **ADA** (το νόμισμα του **Cardano**).
    - **Διαχείριση Πορτοφολιού**: Το πορτοφόλι JetPack επιτρέπει ασφαλή διαχείριση των tokens και μεταφορές σε άλλες υπηρεσίες.

### **Αναμενόμενα Οφέλη από την Εκτίμηση της Αξίας των Token**

- **Ρευστότητα Token**: Η διαπραγμάτευση tokens σε ανταλλακτήρια αυξάνει τη **ρευστότητά** τους.
- **Οικονομικά Οφέλη**: Οι χρήστες μπορούν να επιτύχουν **πρόσθετα οικονομικά οφέλη** πουλώντας tokens αν η αξία τους αυξηθεί.

## Αγορά και Αναδιανομή Token

### **Πρόσθετες Αγορές Token από Υπηρεσίες**

- **Όταν Εξαντληθούν τα Διανεμημένα Tokens**: Αν μια υπηρεσία χρησιμοποιήσει όλα τα διανεμημένα tokens της, μπορεί να αγοράσει επιπλέον tokens από την **Υπηρεσία Ανταλλαγής Πορτοφολιού JetPack**.
- **Νόμισμα Αγοράς**: Τα tokens αγοράζονται χρησιμοποιώντας **ADA**.
- **Χρήση Αγορασμένων Tokens**: Τα αγορασμένα tokens μπορούν να χρησιμοποιηθούν για **ανταμοιβή χρηστών** ή **εντός της υπηρεσίας**.

### **Αναδιανεμημένα έναντι Μη Αναδιανεμημένων Tokens**

- **Διανεμημένα Tokens** (**Μη Αναδιανεμημένα**): Τα tokens που διατίθενται στις υπηρεσίες **δεν μπορούν να αναδιανεμηθούν** και είναι μόνο για **ανταμοιβές χρηστών**.
- **Αγορασμένα Tokens και Tokens που Αποκτήθηκαν από Χρήστες** (**Αναδιανεμημένα**): Τα tokens που αγοράστηκαν με ADA και τα tokens που λήφθηκαν από χρήστες **μπορούν να αναδιανεμηθούν**, βοηθώντας στην **απόκτηση ρευστότητας**.

### **Αναδιανομή Token και Ενεργοποίηση Οικοσυστήματος**

1. **Δομή Κυκλοφορίας Token**:
    - Οι υπηρεσίες ανταμείβουν τους χρήστες με tokens. Οι χρήστες τα χρησιμοποιούν εντός των υπηρεσιών ή τα πωλούν σε ανταλλακτήρια.
    - Οι υπηρεσίες αποκτούν tokens από χρήστες ή τα αγοράζουν για να διατηρήσουν ένα **βιώσιμο σύστημα ανταμοιβών**.

2. **Επιδράσεις Ενεργοποίησης Οικοσυστήματος**:
    - Η κυκλοφορία των tokens προάγει την **εμπλοκή των χρηστών** και την **ενεργοποίηση των υπηρεσιών**.
    - Αυτό συμβάλλει στη **σταθεροποίηση της αξίας των token** και στην **βιώσιμη ανάπτυξη** του οικοσυστήματος.
