# JetPack Token များ အလုပ်လုပ်ပုံ

JetPack token စနစ်တွင် **လှုပ်ရှားမှုအခြေပြု token ထုတ်ပေးခြင်းနှင့် ဖြန့်ဝေခြင်း**, **ဝန်ဆောင်မှုအဆင့်စနစ်**, **အသုံးပြုသူဆုလာဘ်များနှင့် token အသုံးပြုမှုနည်းလမ်းများ**, နှင့် **token များကို ဝယ်ယူခြင်းနှင့် ပြန်လည်ဖြန့်ဝေခြင်း механизм** ပါဝင်သည်။ ဤဖွဲ့စည်းမှုသည် အသုံးပြုသူများနှင့် ဝန်ဆောင်မှုပေးသူများအတွက် **တရားမျှတပြီး ထိရောက်သော ဆုလာဘ်စနစ်** ကို ပေးစွမ်းသည်။

## Token ဖြန့်ဝေခြင်း (Drop) механизм

### စုစုပေါင်း Token ထုတ်ပေးမှုနှင့် ဖြန့်ဝေမှုကာလ

- **စုစုပေါင်းထုတ်ပေးမှု**: JetPack သည် စုစုပေါင်း **1 ဘီလီယံ Jet** token များကို ထုတ်ပေးမည်ဖြစ်သည်။
- **ဖြန့်ဝေမှုကာလ**: Token များကို **38 နှစ်**အတွင်း ဖြန့်ဝေမည်ဖြစ်ပြီး၊ စုစုပေါင်း **2,776 ကြိမ်**ဖြန့်ဝေမှုများကို **Cardano Epoch** (1 epoch = 5 ရက်) နှင့် တူညီသော အချိန်ကာလတွင် ပြုလုပ်မည်ဖြစ်သည်။

### Epoch တစ်ခုစီအတွက် Token ဖြန့်ဝေမှုပမာဏ

#### Logarithmic Function အပေါ်အခြေခံ၍ ဖြန့်ဝေမှုပမာဏတွက်ချက်ခြင်း

Epoch တစ်ခုစီတွင် ဖြန့်ဝေသော token ပမာဏသည် အချိန်ကြာလာသည်နှင့်အမျှ လျော့နည်းသွားရန် ဒီဇိုင်းထုတ်ထားသည်။ **Logarithmic function** ကို အသုံးပြု၍ Epoch တစ်ခုစီအတွက် ဖြန့်ဝေမှုပမာဏကို တွက်ချက်သည်။ **1 Jet သည် 1,000,000 Pack** နှင့် ညီမျှသောကြောင့် ဖြန့်ဝေမှုပမာဏများကို **ခြောက်လုံးစဉ်**အထိ တွက်ချက်ပြီး ပေးဆောင်သည်။

1. **Logarithmic Function $ f(t) $ ၏ သတ်မှတ်ချက်**:

   $$
   f(t) = \dfrac{1}{\ln(t + 1)}
   $$

   အရာ:

    - $ t $ သည် epoch နံပါတ်ဖြစ်ပြီး၊ $ t = 1, 2, 3, \dotsc, 2776 $ ဖြစ်သည်။
    - $ \ln $ သည် သဘာဝလော့ဂရစ် function ဖြစ်သည်။

2. **Epoch အားလုံးအတွက် Log Function $ f(t) $ ၏ စုစုပေါင်းကို တွက်ချက်ခြင်း**:

   ဖြန့်ဝေမှုကာလတစ်ခုလုံးအတွက် function တန်ဖိုးများ၏ စုစုပေါင်း $ S $ သည်:

   $$
   S = \sum_{t=1}^{2776} f(t) = \sum_{t=1}^{2776} \dfrac{1}{\ln(t + 1)}
   $$

3. **Scaling Constant $ A $ ကို တွက်ချက်ခြင်း**:

   **1 ဘီလီယံ Jet** ၏ စုစုပေါင်း token ထုတ်ပေးမှုကို $ S $ ဖြင့် ခွဲ၍ Epoch တစ်ခုစီအတွက် ဖြန့်ဝေမှုပမာဏကို ချိန်ညှိသည်:

   $$
   A = \dfrac{1,000,000,000}{S} = 2,412,005.7263929145
   $$

4. **Epoch တစ်ခုစီအတွက် Token ဖြန့်ဝေမှုပမာဏကို တွက်ချက်ခြင်း**:

   Epoch တစ်ခုစီအတွက် ဖြန့်ဝေမှုပမာဏသည်:

   $$
   \text{Distribution Amount}_t = A \times f(t) = A \times \dfrac{1}{\ln(t + 1)}
   $$

5. **ဖြန့်ဝေမှုပမာဏများ၏ ဥပမာများ**:

   အစောပိုင်း epoch များအတွက် ဖြန့်ဝေမှုပမာဏများကို တွက်ချက်ခြင်း:

    - **Epoch 1**:

      $$
      \text{Distribution Amount}_1 = 3,479,788.70 = A \times \dfrac{1}{\ln(1 + 1)} = A \times \dfrac{1}{\ln(2)}
      $$

    - **Epoch 2**:

      $$
      \text{Distribution Amount}_2 = 2,195,502.227 = A \times \dfrac{1}{\ln(2 + 1)} = A \times \dfrac{1}{\ln(3)}
      $$

    - **Epoch 3**:

      $$
      \text{Distribution Amount}_3 = 1,739,894.35 = A \times \dfrac{1}{\ln(3 + 1)} = A \times \dfrac{1}{\ln(4)}
      $$

   Epoch နံပါတ်နှင့် ကိုက်ညီသော log function ၏ တန်ဖိုးဖြင့် Epoch တစ်ခုစီအတွက် ဖြန့်ဝေမှုပမာဏကို သတ်မှတ်သည်။

**ဖြန့်ဝေမှု၏ လက္ခဏာများ**

- $ \ln(t + 1) $ သည် $ t $ တိုးလာသည်နှင့်အမျှ တိုးလာသောကြောင့် အနိမ့်ဆုံးဖြစ်ပြီး၊ စုစုပေါင်းဖြန့်ဝေမှုပမာဏသည် တဖြည်းဖြည်း လျော့နည်းသွားသည်။
- Epoch တစ်ခုစီအတွက် ဖြန့်ဝေမှုပမာဏသည် အချိန်ကြာလာသည်နှင့်အမျှ လျော့နည်းလာပြီး၊ သုညကို ချဉ်းကပ်လာသည်။
- ဤဒီဇိုင်းသည် အစောပိုင်းပါဝင်သူများကို အားပေးပြီး၊ token ၏ ရှားပါးမှုကို တိုးတက်စေသည်။

### ဝန်ဆောင်မှုအလိုက် Token ဖြန့်ဝေမှု စံနှုန်းများ

#### ဝန်ဆောင်မှုများအကြား အချိုးများကို တွက်ချက်ခြင်း

ဝန်ဆောင်မှုတစ်ခုစီအတွက် ဖြန့်ဝေမှုအချိုးကို ၎င်း၏ စွမ်းဆောင်ရည်အမှတ် $ P_n $ ကို စုစုပေါင်း စွမ်းဆောင်ရည်အမှတ် $ P $ နှင့် နှိုင်းယှဉ်၍ တွက်ချက်သည်:

$$
\text{Ratio for Service } n = \frac{P_n}{P}
$$

အရာ:

- $ P_n $: ဝန်ဆောင်မှု $ n $ ၏ စွမ်းဆောင်ရည်အမှတ်။
- $ P $: ဝန်ဆောင်မှုအားလုံး၏ စွမ်းဆောင်ရည်အမှတ်များ၏ စုစုပေါင်း။

#### ဝန်ဆောင်မှုအလိုက် ဖြန့်ဝေမှုပမာဏကို တွက်ချက်ခြင်း

Epoch တစ်ခုစီတွင် ဝန်ဆောင်မှု $ n $ သည် ရရှိသော token ပမာဏ $ S_n $ သည်:

$$
S_n = \min\left(f_{\text{total}}(t) \times \left( \frac{P_n}{P} \right), \text{Maximum Token Distribution for Service } n\right)
$$

- $ f_{\text{total}}(t) $: Epoch အတွက် ချိန်ညှိထားသော စုစုပေါင်း ဖြန့်ဝေမှုပမာဏ။
- $ \text{Maximum Token Distribution for Service } n $: ဝန်ဆောင်မှု၏ အဆင့်အပေါ်မူတည်၍ သတ်မှတ်ထားသော အများဆုံး token ဖြန့်ဝေမှုပမာဏ (အောက်တွင် အဆင့်အပေါ်မူတည်သော အများဆုံး token ဖြန့်ဝေမှုကို ကြည့်ပါ)။

#### မဖြန့်ဝေသေးသော Token များကို ကိုင်တွယ်ခြင်း

##### မဖြန့်ဝေသေးသော Token များကို ဆက်သွယ်ခြင်း

- Epoch တစ်ခုစီမှ မဖြန့်ဝေသေးသော token များကို နောက်တစ်ခုသော Epoch အတွက် စုစုပေါင်း ဖြန့်ဝေမှုပမာဏတွင် **ထည့်**သည်။
- ဤသည်သည် အစောပိုင်း ဝန်ဆောင်မှု ပါဝင်မှုနည်းပါးသောအခါတွင်တောင် ထိရောက်သော token ဖြန့်ဝေမှုကို သေချာစေသည်။

##### Epoch တစ်ခုစီအတွက် ချိန်ညှိထားသော စုစုပေါင်း ဖြန့်ဝေမှုပမာဏကို တွက်ချက်ခြင်း

နောက်တစ်ခုသော Epoch အတွက် စုစုပေါင်း ဖြန့်ဝေမှုပမာဏသည်:

$$
f_{\text{total}}(t+1) = A \times f(t+1) + \text{Undistributed Tokens from Previous Epoch}
$$

- $ f_{\text{total}}(t+1) $: Epoch $ t+1 $ အတွက် ချိန်ညှိထားသော စုစုပေါင်း ဖြန့်ဝေမှုပမာဏ။
- $ A \times f(t+1) $: အခြေခံ ဖြန့်ဝေမှု function အပေါ်မူတည်သော ဖြန့်ဝေမှုပမာဏ။
- **Carried-over Undistributed Tokens**: Epoch $ t $ တွင် မဖြန့်ဝေသေးသော token များ၏ အရေအတွက်။

#### ဝန်ဆောင်မှုအလိုက် စွမ်းဆောင်ရည်အမှတ် ($ P_n $) ကို တွက်ချက်ခြင်း

ဝန်ဆောင်မှုတစ်ခုစီ၏ စွမ်းဆောင်ရည်အမှတ်ကို အောက်ပါ စံချိန်များအပေါ်မူတည်၍ တွက်ချက်သည်။ စံချိန်တစ်ခုစီ၏ အလေးချိန်ကို အုပ်ချုပ်မှုမှတဆင့် ချိန်ညှိနိုင်သည်။

1. **JetPack Token အသုံးပြုမှု ($ M_{1n} $)**
    - **အလေးချိန်**: $ w_1 = 50\% $
    - **ဖော်ပြချက်**: ဝန်ဆောင်မှုအတွင်း အသုံးပြုသူများမှ အသုံးပြုသော JetPack token များ၏ စုစုပေါင်းပမာဏ။

2. **Degen Bunny မဲရေတွက် ($ M_{2n} $)**
    - **အလေးချိန်**: $ w_2 = 30\% $
    - **ဖော်ပြချက်**: OG Degen Bunnies မှ ရရှိသော မဲရေတွက်။

3. **Active User အရေအတွက် ($ M_{3n} $)**
    - **အလေးချိန်**: $ w_3 = 20\% $
    - **ဖော်ပြချက်**: Epoch အတွင်း ဝန်ဆောင်မှုကို ဝင်ရောက်ပြီး လှုပ်ရှားခဲ့သော ထူးခြားသော အသုံးပြုသူများ၏ အရေအတွက်။

ဝန်ဆောင်မှုတစ်ခုစီ၏ စွမ်းဆောင်ရည်အမှတ် $ P_n $ ကို အောက်ပါအတိုင်း တွက်ချက်သည်:

$$
P_n = w_1 \times \left( \frac{M_{1n}}{M_{1\text{total}}} \right) + w_2 \times \left( \frac{M_{2n}}{M_{2\text{total}}} \right) + w_3 \times \left( \frac{M_{3n}}{M_{3\text{total}}} \right)
$$

အရာ:

- $ M_{1n} $: ဝန်ဆောင်မှု $ n $ ၏ JetPack token အသုံးပြုမှု။
- $ M_{1\text{total}} $: ဝန်ဆောင်မှုအားလုံးအတွက် JetPack token အသုံးပြုမှု စုစုပေါင်း။
- $ M_{2n} $: ဝန်ဆောင်မှု $ n $ ၏ Degen Bunny မဲရေတွက်။
- $ M_{2\text{total}} $: ဝန်ဆောင်မှုအားလုံးအတွက် Degen Bunny မဲရေတွက် စုစုပေါင်း။
- $ M_{3n} $: ဝန်ဆောင်မှု $ n $ ၏ Active User အရေအတွက်။
- $ M_{3\text{total}} $: ဝန်ဆောင်မှုအားလုံးအတွက် Active User အရေအတွက် စုစုပေါင်း။

**ဥပမာ**:

- **ဝန်ဆောင်မှု A**:
    - $ M_{1A} = 500 $
    - $ M_{2A} = 150 $
    - $ M_{3A} = 200 $
- **ဝန်ဆောင်မှု B**:
    - $ M_{1B} = 300 $
    - $ M_{2B} = 200 $
    - $ M_{3B} = 100 $
- **ဝန်ဆောင်မှု C**:
    - $ M_{1C} = 200 $
    - $ M_{2C} = 150 $
    - $ M_{3C} = 700 $

**စုစုပေါင်း**:

- $ M_{1\text{total}} = 500 + 300 + 200 = 1000 $
- $ M_{2\text{total}} = 150 + 200 + 150 = 500 $
- $ M_{3\text{total}} = 200 + 100 + 700 = 1000 $

**အလေးချိန်များ**:

- $ w_1 = 0.5 $
- $ w_2 = 0.3 $
- $ w_3 = 0.2 $

**ဝန်ဆောင်မှု A အတွက် စွမ်းဆောင်ရည်အမှတ်ကို တွက်ချက်ခြင်း**:

$$
\begin{align*}
P_A &= 0.5 \times \left( \frac{500}{1000} \right) + 0.3 \times \left( \frac{150}{500} \right) + 0.2 \times \left( \frac{200}{1000} \right) \\
&= 0.5 \times 0.5 + 0.3 \times 0.3 + 0.2 \times 0.2 \\
&= 0.25 + 0.09 + 0.04 \\
&= 0.38
\end{align*}
$$

2. **ဝန်ဆောင်မှု အဆင့်သတ်မှတ်ပါ**

   စွမ်းဆောင်ရည်အမှတ်များအပေါ်မူတည်၍ ဝန်ဆောင်မှုအားလုံးကို **ဆင်းရဲစဉ်အတိုင်း** အဆင့်သတ်မှတ်ပါ။

3. **စုစုပေါင်း ထိပ်ဆုံးရာခိုင်နှုန်းများကို တွက်ချက်ပါ**

   $$
   \text{Cumulative Top Percentile (\%)} = \left(1 - \frac{\text{Rank} - 1}{N} \right) \times 100\%
   $$

   အရာ:

    - $ \text{Rank} $: ဝန်ဆောင်မှု၏ အဆင့် (1 မှ စတင်)။
    - $ N $: ဝန်ဆောင်မှုများ၏ စုစုပေါင်းအရေအတွက်။

    - **Level 4**: 60% < စုစုပေါင်း ထိပ်ဆုံးရာခိုင်နှုန်း ≤ 77%

    - **Level 8**: 4% < စုစုပေါင်း ထိပ်ဆုံးရာခိုင်နှုန်း ≤ 11%

- Token များကို ဆက်သွယ်ခြင်းသည် **တန်ဖိုးတည်ငြိမ်မှုကို ထိန်းသိမ်းရန်** နှင့် လည်ပတ်မှုကို စီမံခန့်ခွဲရန် အထောက်အကူပြုသည်။

## အသုံးပြုသူဆုလာဘ်များနှင့် Token အသုံးပြုမှု

### **အသုံးပြုသူဆုလာဘ် механизм**

- **လှုပ်ရှားမှုအခြေပြု ဆုလာဘ်များ**: အသုံးပြုသူများသည် ဝန်ဆောင်မှုအတွင်း **ကြိုတင်သဘောတူထားသော လှုပ်ရှားမှုများ** (ဥပမာ၊ အကြောင်းအရာဖန်တီးခြင်း၊ ဂိမ်းကစားခြင်း) ပြုလုပ်သောအခါ JetPack token များကို ရရှိသည်။
- **Token ဆုလာဘ် စံနှုန်းများ**: လှုပ်ရှားမှုတစ်ခုစီအတွက် ဆုလာဘ် token ပမာဏကို ဝန်ဆောင်မှုနှင့် JetPack ecosystem အစီအစဉ်မှူးတို့အကြား **ဆွေးနွေးမှုမှတဆင့် သတ်မှတ်သည်**။

### **Token အသုံးပြုမှု နည်းလမ်းများ**

1. **ဝန်ဆောင်မှုအတွင်း**:
    - **အထူးအင်္ဂါရပ်များကို ဝင်ရောက်ခြင်း**: ဝန်ဆောင်မှုအတွင်း အခပေးအင်္ဂါရပ်များ သို့မဟုတ် အကြောင်းအရာများကို ဝင်ရောက်ရန် token များကို အသုံးပြုပါ။
    - **ကုန်ပစ္စည်းများကို ဝယ်ယူခြင်း**: token များကို အသုံးပြု၍ ဒစ်ဂျစ်တယ်ကုန်ပစ္စည်းများ သို့မဟုတ် ဝန်ဆောင်မှုအတွင်း ပစ္စည်းများကို ဝယ်ယူပါ။

2. **JetPack Wallet Service Exchange ကို အသုံးပြုခြင်း**:
    - **Token ရောင်းချခြင်း**: အသုံးပြုသူများသည် ၎င်းတို့၏ token များကို **ADA** (Cardano ၏ ငွေကြေး) ဖြင့် လဲလှယ်နိုင်သည်။
    - **Wallet စီမံခန့်ခွဲမှု**: JetPack wallet သည် token များကို လုံခြုံစွာ စီမံခန့်ခွဲခြင်းနှင့် အခြားဝန်ဆောင်မှုများသို့ လွှဲပြောင်းခြင်းကို ခွင့်ပြုသည်။

### **Token တန်ဖိုးတိုးတက်မှုမှ မျှော်မှန်းထားသော အကျိုးကျေးဇူးများ**

- **Token ရည်ရွယ်ချက်**: လဲလှယ်မှုများတွင် token များကို လဲလှယ်ခြင်းသည် ၎င်းတို့၏ **ရည်ရွယ်ချက်**ကို တိုးတက်စေသည်။
- **စီးပွားရေး အကျိုးကျေးဇူးများ**: token များ၏ တန်ဖိုးတိုးတက်ပါက ၎င်းတို့ကို ရောင်းချခြင်းဖြင့် အသုံးပြုသူများသည် **အပိုစီးပွားရေး အကျိုးကျေးဇူးများ** ရရှိနိုင်သည်။

## Token ဝယ်ယူခြင်းနှင့် ပြန်လည်ဖြန့်ဝေခြင်း

### **ဝန်ဆောင်မှုများမှ အပို Token ဝယ်ယူခြင်း**

- **ဖြန့်ဝေထားသော Token များ ကုန်ဆုံးသွားသောအခါ**: ဝန်ဆောင်မှုတစ်ခုသည် ၎င်း၏ သတ်မှတ်ထားသော token များအားလုံးကို အသုံးပြုပါက၊ ၎င်းသည် **JetPack Wallet Service Exchange** မှ အပို token များကို ဝယ်ယူနိုင်သည်။
- **ဝယ်ယူငွေကြေး**: Token များကို **ADA** ဖြင့် ဝယ်ယူသည်။
- **ဝယ်ယူထားသော Token များ၏ အသုံးပြုမှု**: ဝယ်ယူထားသော token များကို **အသုံးပြုသူများကို ဆုလာဘ်ပေးရန်** သို့မဟုတ် **ဝန်ဆောင်မှုအတွင်း** အသုံးပြုနိုင်သည်။

### **ပြန်လည်ရောင်းချနိုင်သော Token များနှင့် မရောင်းချနိုင်သော Token များ**

- **ဖြန့်ဝေထားသော Token များ** (**မရောင်းချနိုင်သော**): ဝန်ဆောင်မှုများကို သတ်မှတ်ထားသော token များကို **ပြန်လည်ရောင်းချ၍ မရ**ပြီး၊ ၎င်းတို့ကို **အသုံးပြုသူဆုလာဘ်များအတွက်သာ** ဖြစ်သည်။
- **ဝယ်ယူထားသော Token များနှင့် အသုံးပြုသူများထံမှ ရရှိသော Token များ** (**ပြန်လည်ရောင်းချနိုင်သော**): ADA ဖြင့် ဝယ်ယူထားသော token များနှင့် အသုံးပြုသူများထံမှ ရရှိသော token များကို **ပြန်လည်ရောင်းချနိုင်**ပြီး၊ **ရည်ရွယ်ချက်ရရှိမှု**ကို အထောက်အကူပြုသည်။

### **Token ပြန်လည်ဖြန့်ဝေခြင်းနှင့် Ecosystem လှုပ်ရှားမှု**

1. **Token လည်ပတ်မှု ဖွဲ့စည်းမှု**:
    - ဝန်ဆောင်မှုများသည် အသုံးပြုသူများကို token များဖြင့် ဆုလာဘ်ပေးသည်။ အသုံးပြုသူများသည် ၎င်းတို့ကို ဝန်ဆောင်မှုများအတွင်း အသုံးပြုသည် သို့မဟုတ် လဲလှယ်မှုများတွင် ရောင်းချသည်။
    - ဝန်ဆောင်မှုများသည် အသုံးပြုသူများထံမှ token များကို ရရှိသည် သို့မဟုတ် ဝယ်ယူသည်။ **တည်ငြိမ်သော ဆုလာဘ်စနစ်**ကို ထိန်းသိမ်းရန်။

2. **Ecosystem လှုပ်ရှားမှု အကျိုးသက်ရောက်မှုများ**:
    - Token လည်ပတ်မှုသည် **အသုံးပြုသူ ပါဝင်မှု**နှင့် **ဝန်ဆောင်မှု လှုပ်ရှားမှု**ကို အားပေးသည်။
    - ဤသည်သည် **token တန်ဖိုးတည်ငြိမ်မှု**နှင့် Ecosystem ၏ **တည်ငြိမ်သော တိုးတက်မှု**ကို အထောက်အကူပြုသည်။
