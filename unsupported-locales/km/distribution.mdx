# របៀបដំណើរការរបស់ JetPack Tokens

ប្រព័ន្ធ JetPack token រួមមាន **ការចេញនិងចែកចាយ token ដោយផ្អែកលើសកម្មភាព** មួយប្រព័ន្ធ **កម្រិតសេវាកម្ម** មធ្យោបាយ **រង្វាន់អ្នកប្រើប្រាស់និងការប្រើប្រាស់ token** និងមេកានិចមួយសម្រាប់ **ការទិញនិងចែកចាយ token**។ រចនាសម្ព័ន្ធនេះផ្តល់នូវប្រព័ន្ធរង្វាន់ដែល **សមស្របនិងមានប្រសិទ្ធភាព** សម្រាប់ទាំងអ្នកប្រើប្រាស់និងអ្នកផ្តល់សេវាកម្ម។

## មេកានិចចែកចាយ Token (Drop)

### បរិមាណសរុបនៃ Token និងរយៈពេលចែកចាយ

- **បរិមាណសរុប**: JetPack នឹងចេញ **1 ពាន់លាន Jet** tokens។
- **រយៈពេលចែកចាយ**: Tokens នឹងត្រូវបានចែកចាយក្នុងរយៈពេល **38 ឆ្នាំ** ដោយមានការចែកចាយសរុប **2,776 ដង** ដែលកើតឡើងនៅចន្លោះពេលដូចគ្នានឹង **Cardano Epoch** (1 epoch = 5 ថ្ងៃ)។

### បរិមាណចែកចាយ Token ក្នុងមួយ Epoch

#### គណនាបរិមាណចែកចាយដោយផ្អែកលើមុខងារឡូការីត

បរិមាណនៃ tokens ដែលចែកចាយក្នុងមួយ epoch ត្រូវបានរចនាឡើងដើម្បីបន្ថយទៅតាមពេលវេលា។ មុខងារឡូការីតត្រូវបានប្រើដើម្បីគណនាបរិមាណចែកចាយសម្រាប់មួយៗ epoch។ **1 Jet ស្មើនឹង 1,000,000 Pack** ដូច្នេះបរិមាណចែកចាយត្រូវបានគណនាដល់ **ទីតាំងទសភាគទីប្រាំមួយ** ហើយបង់ប្រាក់តាមសមរម្យ។

1. **និយមន័យនៃមុខងារឡូការីត $ f(t) $**:

   $$
   f(t) = \dfrac{1}{\ln(t + 1)}
   $$

   ដែលនៅទីនេះ:

    - $ t $ គឺជាលេខ epoch, $ t = 1, 2, 3, \dotsc, 2776 $។
    - $ \ln $ គឺជាមុខងារឡូការីតធម្មជាតិ។

2. **គណនាផលបូកនៃមុខងារឡូការីត $ f(t) $ លើគ្រប់ Epochs**:

   ផលបូកសរុបនៃតម្លៃមុខងារនៅលើរយៈពេលចែកចាយទាំងមូល $ S $ គឺជា:

   $$
   S = \sum_{t=1}^{2776} f(t) = \sum_{t=1}^{2776} \dfrac{1}{\ln(t + 1)}
   $$

3. **គណនាតម្លៃថេរតុល្យភាព $ A $**:

   បរិមាណសរុបនៃ token **1 ពាន់លាន Jet** ត្រូវបានបែងចែកដោយផលបូក $ S $ ដើម្បីកែតម្រូវបរិមាណចែកចាយសម្រាប់មួយៗ epoch:

   $$
   A = \dfrac{1,000,000,000}{S} = 2,412,005.7263929145
   $$

4. **គណនាបរិមាណចែកចាយ Token ក្នុងមួយ Epoch**:

   បរិមាណចែកចាយសម្រាប់មួយៗ epoch គឺជា:

   $$
   \text{Distribution Amount}_t = A \times f(t) = A \times \dfrac{1}{\ln(t + 1)}
   $$

5. **ឧទាហរណ៍នៃបរិមាណចែកចាយ**:

   គណនាបរិមាណចែកចាយសម្រាប់ epoch ដំបូងៗ:

    - **Epoch 1**:

      $$
      \text{Distribution Amount}_1 = 3,479,788.70 = A \times \dfrac{1}{\ln(1 + 1)} = A \times \dfrac{1}{\ln(2)}
      $$

    - **Epoch 2**:

      $$
      \text{Distribution Amount}_2 = 2,195,502.227 = A \times \dfrac{1}{\ln(2 + 1)} = A \times \dfrac{1}{\ln(3)}
      $$

    - **Epoch 3**:

      $$
      \text{Distribution Amount}_3 = 1,739,894.35 = A \times \dfrac{1}{\ln(3 + 1)} = A \times \dfrac{1}{\ln(4)}
      $$

   បរិមាណចែកចាយសម្រាប់មួយៗ epoch ត្រូវបានកំណត់ដោយតម្លៃនៃមុខងារឡូការីតដែលសមស្របនឹងលេខ epoch។

**លក្ខណៈពិសេសនៃការចែកចាយ**

- ពីព្រោះ $ \ln(t + 1) $ កើនឡើងនៅពេល $ t $ កើនឡើង, តម្លៃបែងចែកក្លាយជាធំជាង, បណ្តាលឱ្យបរិមាណចែកចាយសរុបបន្ថយទៅតិចៗ។
- បរិមាណចែកចាយក្នុងមួយ epoch បន្ថយទៅតិចៗ, ឈានដល់សូន្យ។
- ការរចនានេះលើកទឹកចិត្តអ្នកចូលរួមដំបូងនិងបង្កើនភាពកម្ររបស់ token។

### លក្ខណៈនៃការចែកចាយ Token ក្នុងមួយសេវាកម្ម

#### គណនាអនុបាតរវាងសេវាកម្ម

អនុបាតចែកចាយសម្រាប់មួយៗសេវាកម្មត្រូវបានគណនាដោយផ្អែកលើពិន្ទុសមត្ថភាព $ P_n $ របស់វាដែលទាក់ទងនឹងពិន្ទុសមត្ថភាពសរុប $ P $:

$$
\text{Ratio for Service } n = \frac{P_n}{P}
$$

ដែលនៅទីនេះ:

- $ P_n $: ពិន្ទុសមត្ថភាពនៃសេវាកម្ម $ n $។
- $ P $: ផលបូកនៃពិន្ទុសមត្ថភាពនៃគ្រប់សេវាកម្ម។

#### គណនាបរិមាណចែកចាយសម្រាប់មួយៗសេវាកម្ម

បរិមាណនៃ tokens ដែលសេវាកម្ម $ n $ ទទួលបានក្នុងមួយ epoch $ S_n $ គឺជា:

$$
S_n = \min\left(f_{\text{total}}(t) \times \left( \frac{P_n}{P} \right), \text{Maximum Token Distribution for Service } n\right)
$$

- $ f_{\text{total}}(t) $: បរិមាណចែកចាយសរុបដែលបានកែតម្រូវសម្រាប់ epoch។
- $ \text{Maximum Token Distribution for Service } n $: បរិមាណ token អតិបរមាដែលបានចាត់តាមកម្រិតនៃសេវាកម្ម (មើលការចែកចាយ token អតិបរមាតាមកម្រិតខាងក្រោម)។

#### ការដោះស្រាយនៃ Tokens មិនបានចែកចាយ

##### ការបន្ត Tokens មិនបានចែកចាយ

- Tokens មិនបានចែកចាយពីមួយៗ epoch ត្រូវបាន **បន្ថែម** ទៅបរិមាណចែកចាយសរុបសម្រាប់ epoch បន្ទាប់។
- នេះធានាថាការចែកចាយ token មានប្រសិទ្ធភាពទោះបីជាការចូលរួមសេវាកម្មដំបូងមានតិច។

##### គណនាបរិមាណចែកចាយសរុបដែលបានកែតម្រូវក្នុងមួយ Epoch

បរិមាណចែកចាយសរុបសម្រាប់ epoch បន្ទាប់គឺជា:

$$
f_{\text{total}}(t+1) = A \times f(t+1) + \text{Undistributed Tokens from Previous Epoch}
$$

- $ f_{\text{total}}(t+1) $: បរិមាណចែកចាយសរុបដែលបានកែតម្រូវសម្រាប់ epoch $ t+1 $។
- $ A \times f(t+1) $: បរិមាណចែកចាយដោយផ្អែកលើមុខងារចែកចាយមូលដ្ឋាន។
- **Tokens មិនបានចែកចាយដែលបានបន្ត**: ចំនួន tokens មិនបានចែកចាយក្នុង epoch $ t $។

#### គណនាពិន្ទុសមត្ថភាព ($ P_n $) សម្រាប់មួយៗសេវាកម្ម

ពិន្ទុសមត្ថភាពនៃមួយៗសេវាកម្មត្រូវបានគណនាដោយផ្អែកលើមាត្រដ្ឋានខាងក្រោម។ ទម្ងន់នៃមាត្រដ្ឋានមួយៗអាចត្រូវបានកែតម្រូវតាមរដ្ឋាភិបាល។

1. **ការប្រើប្រាស់ JetPack Token ($ M_{1n} $)**
    - **ទម្ងន់**: $ w_1 = 50\% $
    - **ការពិពណ៌នា**: បរិមាណសរុបនៃ JetPack tokens ដែលបានប្រើប្រាស់ដោយអ្នកប្រើប្រាស់ក្នុងសេវាកម្ម។

2. **ចំនួនបោះឆ្នោត Degen Bunny ($ M_{2n} $)**
    - **ទម្ងន់**: $ w_2 = 30\% $
    - **ការពិពណ៌នា**: ចំនួនឆ្នោតដែលបានទទួលពី OG Degen Bunnies។

3. **ចំនួនអ្នកប្រើប្រាស់សកម្ម ($ M_{3n} $)**
    - **ទម្ងន់**: $ w_3 = 20\% $
    - **ការពិពណ៌នា**: ចំនួនអ្នកប្រើប្រាស់តែមួយដែលបានចូលនិងសកម្មក្នុងសេវាកម្មក្នុងរយៈពេល epoch។

ពិន្ទុសមត្ថភាពនៃមួយៗសេវាកម្ម $ P_n $ ត្រូវបានគណនាដូចខាងក្រោម:

$$
P_n = w_1 \times \left( \frac{M_{1n}}{M_{1\text{total}}} \right) + w_2 \times \left( \frac{M_{2n}}{M_{2\text{total}}} \right) + w_3 \times \left( \frac{M_{3n}}{M_{3\text{total}}} \right)
$$

ដែលនៅទីនេះ:

- $ M_{1n} $: ការប្រើប្រាស់ JetPack token នៃសេវាកម្ម $ n $។
- $ M_{1\text{total}} $: ការប្រើប្រាស់ JetPack token សរុបនៅគ្រប់សេវាកម្ម។
- $ M_{2n} $: ចំនួនបោះឆ្នោត Degen Bunny សម្រាប់សេវាកម្ម $ n $។
- $ M_{2\text{total}} $: ចំនួនបោះឆ្នោត Degen Bunny សរុបនៅគ្រប់សេវាកម្ម។
- $ M_{3n} $: ចំនួនអ្នកប្រើប្រាស់សកម្មនៃសេវាកម្ម $ n $។
- $ M_{3\text{total}} $: ចំនួនអ្នកប្រើប្រាស់សកម្មសរុបនៅគ្រប់សេវាកម្ម។

**ឧទាហរណ៍**:

- **សេវាកម្ម A**:
    - $ M_{1A} = 500 $
    - $ M_{2A} = 150 $
    - $ M_{3A} = 200 $
- **សេវាកម្ម B**:
    - $ M_{1B} = 300 $
    - $ M_{2B} = 200 $
    - $ M_{3B} = 100 $
- **សេវាកម្ម C**:
    - $ M_{1C} = 200 $
    - $ M_{2C} = 150 $
    - $ M_{3C} = 700 $

**សរុប**:

- $ M_{1\text{total}} = 500 + 300 + 200 = 1000 $
- $ M_{2\text{total}} = 150 + 200 + 150 = 500 $
- $ M_{3\text{total}} = 200 + 100 + 700 = 1000 $

**ទម្ងន់**:

- $ w_1 = 0.5 $
- $ w_2 = 0.3 $
- $ w_3 = 0.2 $

**គណនាពិន្ទុសមត្ថភាពសម្រាប់សេវាកម្ម A**:

$$
\begin{align*}
P_A &= 0.5 \times \left( \frac{500}{1000} \right) + 0.3 \times \left( \frac{150}{500} \right) + 0.2 \times \left( \frac{200}{1000} \right) \\
&= 0.5 \times 0.5 + 0.3 \times 0.3 + 0.2 \times 0.2 \\
&= 0.25 + 0.09 + 0.04 \\
&= 0.38
\end{align*}
$$

**គណនាពិន្ទុសមត្ថភាពសម្រាប់សេវាកម្ម B**:

$$
\begin{align*}
P_B &= 0.5 \times \left( \frac{300}{1000} \right) + 0.3 \times \left( \frac{200}{500} \right) + 0.2 \times \left( \frac{100}{1000} \right) \\
&= 0.5 \times 0.3 + 0.3 \times 0.4 + 0.2 \times 0.1 \\
&= 0.15 + 0.12 + 0.02 \\
&= 0.29
\end{align*}
$$

**គណនាពិន្ទុសមត្ថភាពសម្រាប់សេវាកម្ម C**:

&= 0.10 + 0.09 + 0.14 \\
&= 0.33
\end{align*}
$$

- **សេវាកម្ម A**: $ P_A = 0.38 $

##### វិធីសាស្រ្តកំណត់កម្រិត

1. **គណនាពិន្ទុសមត្ថភាព**

   ប្រើវិធីសាស្រ្តដែលបានកំណត់មុននេះដើម្បីគណនាពិន្ទុសមត្ថភាពនៃមួយៗសេវាកម្ម $ P_n $។

2. **កំណត់ចំណាត់ថ្នាក់សេវាកម្ម**

   ចាត់ថ្នាក់គ្រប់សេវាកម្មតាមលំដាប់ **ចុះចត** ដោយផ្អែកលើពិន្ទុសមត្ថភាពរបស់ពួកវា។

3. **គណនាភាគរយកំពូលសរុប**

   សម្រាប់មួយៗសេវាកម្មគណនាភាគរយកំពូលសរុប:

   $$
   \text{Cumulative Top Percentile (\%)} = \left(1 - \frac{\text{Rank} - 1}{N} \right) \times 100\%
   $$

   ដែលនៅទីនេះ:

    - $ \text{Rank} $: ចំណាត់ថ្នាក់នៃសេវាកម្ម (ចាប់ពី 1)។
    - $ N $: ចំនួនសេវាកម្មសរុប។

4. **ចាត់កម្រិត**

   ចាត់កម្រិតដោយផ្អែកលើភាគរយកំពូលសរុប:

    - **កម្រិត 3**: 77% < ភាគរយកំពូលសរុប ≤ 89%

    - **កម្រិត 6**: 23% < ភាគរយកំពូលសរុប ≤ 40%
    - **កម្រិត 7**: 11% < ភាគរយកំពូលសរុប ≤ 23%

### **មេកានិចរង្វាន់អ្នកប្រើប្រាស់**

- **រង្វាន់ដោយផ្អែកលើសកម្មភាព**: អ្នកប្រើប្រាស់ទទួលបាន JetPack tokens នៅពេលពួកគេធ្វើ **សកម្មភាពដែលបានตกลงជាមុន** (ឧទាហរណ៍, ការបង្កើតមាតិកា, ការលេងហ្គេម) ក្នុងសេវាកម្ម។
- **លក្ខណៈរង្វាន់ Token**: បរិមាណនៃ tokens ដែលបានផ្តល់រង្វាន់សម្រាប់មួយៗសកម្មភាពត្រូវបាន **កំណត់តាមការពិភាក្សា** រវាងសេវាកម្មនិងអ្នកប្រតិបត្តិ JetPack ecosystem។

### **វិធីសាស្រ្តប្រើប្រាស់ Token**

1. **ក្នុងសេវាកម្ម**:
    - **ចូលប្រើលក្ខណៈពិសេសពិសេស**: ប្រើ tokens ដើម្បីចូលប្រើលក្ខណៈពិសេសដែលត្រូវបង់ប្រាក់ឬមាតិកាក្នុងសេវាកម្ម។
    - **ទិញទំនិញ**: ទិញទំនិញឌីជីថលឬធាតុក្នុងសេវាកម្មដោយប្រើ tokens។

2. **ប្រើប្រាស់ JetPack Wallet Service Exchange**:
    - **ការលក់ Token**: អ្នកប្រើប្រាស់អាចប្តូរទៅជា **ADA** (រូបិយប័ណ្ណនៃ **Cardano**)។
    - **ការគ្រប់គ្រង Wallet**: JetPack wallet អនុញ្ញាតឱ្យគ្រប់គ្រង tokens យ៉ាងសុវត្ថិភាពនិងផ្ទេរទៅសេវាកម្មផ្សេងទៀត។

### **អត្ថប្រយោជន៍ដែលរំពឹងទុកពីការកើនឡើងតម្លៃ Token**

- **សាច់ប្រាក់ Token**: ការជួញដូរ tokens លើទីផ្សារកើនឡើង **សាច់ប្រាក់** របស់ពួកវា។
- **ប្រាក់ចំណេញសេដ្ឋកិច្ច**: អ្នកប្រើប្រាស់អាចទទួលបាន **អត្ថប្រយោជន៍សេដ្ឋកិច្ចបន្ថែម** ដោយលក់ tokens ប្រសិនបើតម្លៃរបស់ពួកវាកើនឡើង។

## ការទិញនិងចែកចាយ Token

### **ការទិញ Token បន្ថែមដោយសេវាកម្ម**

- **នៅពេល Tokens ដែលបានចែកចាយត្រូវបានប្រើអស់**: ប្រសិនបើសេវាកម្មប្រើប្រាស់ tokens ដែលបានចែកចាយទាំងអស់វាអាចទិញ tokens បន្ថែមពី **JetPack Wallet Service Exchange**។
- **រូបិយប័ណ្ណទិញ**: Tokens ត្រូវបានទិញដោយប្រើ **ADA**។
- **ការប្រើប្រាស់ Tokens ដែលបានទិញ**: Tokens ដែលបានទិញអាចត្រូវបានប្រើដើម្បី **ផ្តល់រង្វាន់អ្នកប្រើប្រាស់** ឬ **ក្នុងសេវាកម្ម**។

### **Tokens ដែលអាចលក់ឡើងវិញនិងមិនអាចលក់ឡើងវិញ**

- **Tokens ដែលបានចែកចាយ** (**មិនអាចលក់ឡើងវិញ**): Tokens ដែលបានចាត់ទៅសេវាកម្ម **មិនអាចលក់ឡើងវិញ** និងត្រូវបានប្រើសម្រាប់ **រង្វាន់អ្នកប្រើប្រាស់**។
- **Tokens ដែលបានទិញនិង Tokens ដែលទទួលបានពីអ្នកប្រើប្រាស់** (**អាចលក់ឡើងវិញ**): Tokens ដែលបានទិញដោយ ADA និង tokens ដែលទទួលបានពីអ្នកប្រើប្រាស់ **អាចលក់ឡើងវិញ** ជួយក្នុងការទទួលបាន **សាច់ប្រាក់**។

### **ការចែកចាយ Token និងការបង្កើនសកម្មភាពនៃ Ecosystem**

1. **រចនាសម្ព័ន្ធសាច់ប្រាក់ Token**:
    - សេវាកម្មផ្តល់រង្វាន់អ្នកប្រើប្រាស់ដោយ tokens; អ្នកប្រើប្រាស់ប្រើប្រាស់វាក្នុងសេវាកម្មឬលក់វានៅលើទីផ្សារ។
    - សេវាកម្មទទួលបាន tokens ពីអ្នកប្រើប្រាស់ឬទិញវាដើម្បីរក្សាប្រព័ន្ធរង្វាន់ដែល **មាននិរន្តរភាព**។

2. **ផលប៉ះពាល់នៃការបង្កើនសកម្មភាពនៃ Ecosystem**:
    - សាច់ប្រាក់ Token លើកទឹកចិត្ត **ការចូលរួមអ្នកប្រើប្រាស់** និង **ការបង្កើនសកម្មភាពសេវាកម្ម**។
    - នេះរួមចំណែកក្នុងការរក្សា **ស្ថិរភាពតម្លៃ Token** និង **ការលូតលាស់ដែលមាននិរន្តរភាព** នៃ ecosystem។
